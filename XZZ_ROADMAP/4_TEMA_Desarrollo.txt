    Relación Frontend y Backend
      Arquitectura Cliente-Servidor
        Frontend: Cliente
        Backend: Servidor
      Flujo de Comunicación
        Usuario interactúa con Frontend
        Frontend hace peticiones al Backend
        Backend procesa y devuelve datos
        Frontend muestra la respuesta
      Separación de Responsabilidades
        Frontend: Presentación
        Backend: Lógica y datos
      Ventajas de la Separación
        Desarrollo paralelo
        Escalabilidad independiente
        Mantenimiento más fácil
    Full Stack Developer
      Definición
        Desarrollador que trabaja en Frontend y Backend
        "Unicornio" del desarrollo
      Habilidades Técnicas Requeridas
        Frontend: HTML, CSS, JavaScript
        Backend: Lenguaje de servidor, BD
        DevOps básico
      Conocimientos Adicionales
        Control de versiones (Git)
        APIs RESTful
        Seguridad web
        Performance optimization
      Perfil del Desarrollador
        Versátil y adaptable
        Visión global del proyecto
        Buen comunicador
    Comunicación API + Full Stack
      ¿Qué es una API?
        Interfaz de Programación de Aplicaciones
        Contrato entre frontend y backend
      Tipos de APIs
        REST
        GraphQL
        SOAP
      Comunicación HTTP
        Métodos: GET, POST, PUT, DELETE
        Status codes: 200, 404, 500
        Headers y body
      Formato de Datos
        JSON (más común)
        XML
      Flujo Completo
        Frontend → Petición HTTP → Backend
        Backend → Procesa → Base de Datos
        Backend → Respuesta JSON → Frontend
        Frontend → Renderiza datos → Usuario
